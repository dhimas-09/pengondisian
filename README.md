penjelasan dan fungsi
1. git init
Penjelasan: Perintah untuk menginisialisasi atau membuat repositori Git baru di direktori lokal komputer kamu.

2. git status
Penjelasan: Digunakan untuk melihat kondisi terkini dari "working directory" (folder kerja) kamu.

3. git add 
penjelasan : untuk memperbaharui atau mengoneksikan, mengsinkronisasikan local dan repository yang ada git commit, mencatat hasil akhir  

4. git commit
Penjelasan: Perintah untuk menyimpan perubahan yang telah kamu tandai (staged) ke dalam sejarah permanen Git. Biasanya disertai dengan pesan menggunakan flag

5. git push origin main
Penjelasan: Mengirimkan (upload) semua commit yang ada di komputer lokal kamu ke repositori daring (remote) seperti GitHub atau GitLab.

Cheat Sheet

1. Repository (repo)
Tempat penyimpanan project (kode + riwayat perubahan).
2. Commit
Menyimpan perubahan yang sudah kamu buat ke repo (seperti “save point”).
3. Hash
Kode unik untuk setiap commit (ID yang membedakan satu commit dengan lainnya).
4. Checkout
Berpindah ke branch atau commit tertentu.
5. Branch (bukan “breanch”)
Cabang pengembangan, biasanya untuk fitur baru tanpa mengganggu kode utama.
6. Merge
Menggabungkan perubahan dari satu branch ke branch lain.
7. Remote
Repo yang ada di server (misalnya di GitHub).
8. Clone
Menyalin repo dari remote ke komputer lokal.
9. Push
Mengirim perubahan dari lokal ke remote.
10. Pull
Mengambil perubahan terbaru dari remote ke lokal.
